Ping Identity Mt Fuji - Web Developer

Congrats! We know you know at least enough about maven to get yourself in trouble. But you knew we knew you knew that.

This programming exercise, or Mt Fuji, is the DevOps team way of seeing how you code. This consists of two assignments, one asking for a JavaScript example, and the other a Java example.

How your code will be evaluated

  • Readability
  • Extensibility
  • Correctness
  • Completeness

As a general rule, programming at Ping prefers simple over complex, and readability over conciseness.

Assignment #1 - JavaScript

Refer to ui-programming-exercise.html

We would like you to write a JS that will position dynamically the help box. You can use any library in the src/main/webapp/assets/ directory -- JQuery/Bootstrap and add on top of it or write a JS from scratch.

Scenario: Our eccentric UI designer from eastern europe has provided a composition, which is assigned to you to implement. You receieved the following screenshots describing expected behavior:

  • 01.jpeg is the screenshot of the actual ui-programming-exercise.html. Please note the way help section is working.
  • 02.jpeg is where you can see the variables -- all are space related.
  • 03.jpeg and 04.jpeg is the expected behavior.

In addition, you've been provided the following instructions:

            Explanation:

            var_window = dynamic value, as we drag and expand/contract the window, this changes
            var_top = dynamic -- accommodate the header elements (if we add more there, this will change)
            var_top_min = 100px -- fixed, define it in the properties of the JS
            var_bottom = 100px -- fixed, define it in the properties of the JS
            var_space = dynamic -- calculate it: var_window -- (var_top/"or var_top_min" + var_bottom)
            var_help = dynamic, but not lower than 355px -- define it in the properties of the JS
                 (now is in the inline style of the element)
        
            Behavior we want:

            Help section (var_help element) shall be positioned static so if the page is expanded 
            (no scroll) or filled with content (scroll) the help will stay in the middle of the 
            var_space element, filling it -- see picture 03.jpeg. 

            If we contract the browser window, the help shall not shrink lower than 355px.

            Now there is a tricky part we want to see. 
            If the page scrolls, and we scroll down so the header is out, the help shall stay 
            positioned to the top by var_top_min, 100px -- see picture 04.jpeg.  
            If we scroll up and the header is appearing, the help shall stop to the var_top and of course, 
            not overlap that area -- see picture 03.jpeg.

            This shall work in latest Firefox, Chrome and Safari, and all IE starting with IE7.

Assignment #2 - Java

At Ping Identity, most of our web development uses Apache Wicket as our MVC framework. Knowing such an esoteric framework isn't necessary for the Mt Fuji, but it can't hurt. Here's how it integrates into our example Spring / Hibernate stack:

Wicket Demo

Scenario: Our eccentric european has again provided a comp for you to implement. However, this one needs to save off to the database, the client side is fully implemented.

In your favorite MVC framework, save off these inputs into the database! We've provided the database calls, as well as a service layer call along with a data transfer object:

  • src/main/java/com/pingone/fuji/web/dto/UserDto.java
  • src/main/java/com/pingone/fuji/wev/svc/FujiSvc.java#saveUser(UserDto userDto)

Using the mockup [1] provided, save the data from the browser into the database. Then using the other mockup[2], load it from the database and display the information saved.

Remember! Be sure to add validation for the fields as indicated by the mockup. We also really, really like unit tests.

To reiterate, you can do this in any MVC framework you want. Wicket integration with Spring/Hibernate is provided.
