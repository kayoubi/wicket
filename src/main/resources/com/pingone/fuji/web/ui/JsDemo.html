About this section

This is the section of the application where you setup your IdentityBridge.

IdentityBridge

An IdentityBridge is a tool that connects authoritative user repositories to cloud infrastructure. By securely connecting user repositories, organizations can store passwords in one place but leverage them to protect many different applications.

In order to connect your users to cloud applications, we must first authenticate those users. Some organizations keep their user information on-premise, in repositories like Active Directory, other organizations use cloud repositories like Google Apps, or infrastructure services like SAML.

PingOne supports many methods of bridging identities into the cloud; choose the method that best suits your organization.

PingOne AD Connect

PingOne AD Connect is a free software tool that can be downloaded and deployed locally on a Windows server so that Active Directory identities can be bridged into CAS. If you don't know which option to pick, and you have Active Directory in your organization, this is your best bet.

For more information on what it means to install AD Connect in your organization, you can watch a quick video here.

Ping Federate (SAML)

PingFederate is a powerful federation server that can send SAML assertions to PingOne (among many other things). Choose this option if you are a PingFederate administrator.

For more information please check out the PingFederate product page.

Google Apps

Google Apps for your Domain is a cloud-based business suite that can generate OpenID assertions and send them to PingOne. If you are already using Google Apps for mail and calendar, there are just a few extra steps you must take to also use Google Apps as your Identity Provider.

For more information please check Google Apps official page.

3rd-party SAML

There are many federation tools that can generate SAML assertions and send them to PingOne. Examples of 3rd Party SAML tools are ADFS, Shibboleth, CA Siteminder, Oracle Identity Federation, and many others. PingOne can accept SAML 2.0 assertions from all of these providers.

3-Party OpenToken IdP

Plug in your own authenticator to PingOne using OpenToken.

If you want to hack some new ways of authenticating people, this is the right choice for you. How about authenticating user through "face recognition"?

Cloud Directory

PingOne Cloud Directory is a powerful cloud based directory server that allows you manage and authenticate your users in a highly secure and extensible infrastructrue. Stop worrying about scalibility, geo-dedundency, availibility, here is your one stop shop to get it all!

For more information please check out the Cloud Direcotry product page.

PingOne Super IdP

Use the Web Portal as your IdP in hacker's mode.

Since we know you are the administrator of your account, we will give you the power to customize the identity attributes (subjects, assertions, claims, anything you can think of) to be sent to PingOne Token Processor, and by doing so, you can impersonate any users under your account.
